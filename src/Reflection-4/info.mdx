# Reflection:
A process where executed result are cross checked by another agent over and over until getting a satisfactory result

## Process:
1. Execution -> initial result getting executed by main agent pipeline
2. Evaluation -> Checker agent checks the main result
3. Reflection -> Based on evaluation, result get improved
4. Iteration -> The whole process of 2 and 3 runs over and over until getting a satisfactory result

## Cons:
It consume too much computation and needs too much token. 

## Use Cases :

- Creative Writing and Content Generation.
- Code Generation and Debugging
- Complex Problem Solving
- Summarization and Information Synthesis
- Planning and Strategy
- Conversational Agents
